Enhancement 3: GrocConnect Databases

Overview

For the database enhancement, there was no artifact from a previous project or coursework. The enhancement is an extension of my first enhancement, which is the MongoDB connection in my Grocconnect web stack application. It establishes a vital connection between the Express server and the database, enabling the application to handle GET, POST, PUT, DEL, etc., requests through specific API endpoints. This enhancement was created when I started my Grocconnect application.

Justify Inclusion

I selected this item for my ePortfolio because it demonstrates both my understanding of backend development and my ability to implement scalable applications in a full-stack environment. The artifact highlights several skills in software development, including API design, server-database interaction through interceptors for authentication, and HTTPRouter. It is also an implementation of a RESTful architecture, which is vital in web development. By enabling dynamic data handling between the client and server, this enhancement improves the overall functionality and responsiveness of the application. The program from category 1 has been improved through clear and secure routes and efficient data schemas using Mongoose. It showcases my ability to move beyond static content delivery and into dynamic, data-driven development.

Additionally, it demonstrates my secure coding mindset. Sensitive information such as the JWT_SECRET encryption string, is stored securely in a .env file, which is then protected through .gitignore practices. User authentication is required to perform CRUD operations on the database, implemented through JSON Web Tokens (JWTs) and stored in the browser using the Local Storage API. I chose Local Storage because it allows for flexible key-value pairing, which fits the needs of this application. Although Local Storage is susceptible to script-injection (XSS) attacks, I recognized this trade-off after evaluating the alternative of using browser cookies, prioritizing ease of access for this project.

Course Outcomes

Yes, the course outcome I planned to meet with this enhancement, the fifth course outcome, was successfully achieved. I implemented a JWT system that generates a token when a user either registers or logs into the admin page. This token is then required for server-side API calls, ensuring that certain HTTP verbs, particularly those that modify data, are protected and only accessible to authenticated users. This prevents unauthorized changes and secures the backend operations.

Additionally, this enhancement also supports the first course outcome. Throughout the project, I incorporated extensive inline comments and informative log messages for debugging purposes. These elements not only assist during development but also provide clarity for future developers by indicating which parts of the codebase are being accessed and how the application flows. As a result, the codebase remains maintainable, understandable, and professional.

Reflection

Enhancing and modifying this artifact was a valuable learning experience that deepened my understanding of full-stack web development, particularly in connecting a front-end interface to a back-end database using MongoDB and Express. As I built and improved the database integration within my Grocconnect application, I gained hands-on experience in designing RESTful APIs, structuring database schemas, and ensuring smooth communication between the server and the database. I am still learning ways to enhance the code base, like introducing a shopping cart where the guest user would be able to purchase the items if implemented for a grocery store. There is still a lot to learn in connecting the backend to the frontend servers of Express or rapid rendering, like with Angular.
